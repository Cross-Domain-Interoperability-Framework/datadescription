# AGENTS.md - AI Agent Guidance for CDIF Data Description

## Project context

This repository hosts the CDIF Data Description profile — extends Discovery with DDI-CDI variable documentation, physical data structure mapping, and file characterization. Records here must use properties beyond what Discovery provides.

## Classification rule

An example belongs here ONLY if it uses at least one of these DataDescription-specific properties:
- `cdi:physicalDataType` or `cdi:intendedDataType` on variableMeasured items
- `cdi:role` (MeasureComponent, AttributeComponent, DimensionComponent, etc.)
- `cdi:hasPhysicalMapping` on distributions (column-to-variable linkage)
- `cdi:TabularTextDataSet`, `cdi:LongStructureDataSet`, or `cdi:StructuredDataSet` distribution types
- `csvw:` properties (delimiter, header, headerRowCount)
- `cdifq:nRows`, `cdifq:nColumns`

Records that only have `cdi:InstanceVariable` in `@type` without any of the above should go to:
- **Discovery** repo — if they have spatialCoverage/temporalCoverage/variableMeasured
- **packaging** repo — if they are archive bundles (hasPart with multiple files)

## Example sources

### DDI Codebook conversion (primary source)

Harvard Dataverse provides DDI Codebook 2.5 with variable-level descriptions for ingested .tab files:

```bash
# Download DDI
curl -o dataset.xml "https://dataverse.harvard.edu/api/datasets/export?exporter=ddi&persistentId=doi:10.7910/DVN/XXX"

# Convert to CDIF DataDescription with physical mappings, file metadata
python validation/DDI/ddi_to_cdif.py dataset.xml \
  --doi https://doi.org/10.7910/DVN/XXX \
  --fetch-headers --fetch-file-meta \
  -o examples/ddi-my-dataset-datadesc.json
```

The DDI-to-CDIF mapping:
- DDI `<var name>` → `schema:name`
- DDI `<labl>` → `schema:description`
- DDI `<varFormat type>` → `cdi:intendedDataType` (numeric→xsd:decimal, character→xsd:string)
- DDI `<var intrvl>` → `cdi:role` (contin→MeasureComponent, discrete→AttributeComponent)
- DDI `<sumStat min/max>` → `schema:minValue`/`schema:maxValue`
- DDI `<caseQnty>` → `cdifq:nRows`
- DDI `<varQnty>` → `cdifq:nColumns`
- Tab file headers → `cdi:hasPhysicalMapping` with column index and `cdi:formats_InstanceVariable`
- Dataverse file API → `schema:contentSize`, `spdx:checksum`

## Validation

```bash
python -c "
import json
from jsonschema import Draft202012Validator
schema = json.load(open('CDIFDataDescriptionProfileStructuredSchema.json'))
doc = json.load(open('examples/ddi-cost-of-cash-datadesc.json'))
errors = list(Draft202012Validator(schema).iter_errors(doc))
print(f'{len(errors)} errors')
"
```

## Key namespaces

| Prefix | IRI | Usage |
|--------|-----|-------|
| `cdi` | `http://ddialliance.org/Specification/DDI-CDI/1.0/RDF/` | InstanceVariable, physicalDataType, role, hasPhysicalMapping, TabularTextDataSet |
| `csvw` | `http://www.w3.org/ns/csvw#` | delimiter, header, headerRowCount |
| `cdifq` | `http://crossdomaininteroperability.org/cdifq/` | nRows, nColumns (placeholder namespace) |
| `spdx` | `http://spdx.org/rdf/terms#` | checksum, algorithm, checksumValue |

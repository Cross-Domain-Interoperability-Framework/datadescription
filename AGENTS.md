# AGENTS.md - AI Agent Guidance for CDIF Data Description

## Project context

This repository hosts the CDIF Data Description profile — extends Discovery with DDI-CDI variable documentation, physical data structure mapping, and file characterization. Records here must use properties beyond what Discovery provides.

## Classification rule

An example belongs here ONLY if it uses at least one of these DataDescription-specific properties:
- `cdif:physicalDataType` or `cdi:hasIntendedDataType` on variableMeasured items
- `cdif:role` (UnitIdentifier, Measure, Attribute, Dimension, Descriptor, ReferenceVariable)
- `cdi:takesSentinelValuesFrom` / `cdi:takesSubstantiveValuesFrom` value-domain links (these are Data-Description-only — Discovery's plain PropertyValue does not carry them)
- `cdif:hasPrimaryKey`, `cdif:statistics`, or `cdif:isDescribedBy_StatisticsCollection`
- `cdif:hasPhysicalMapping` on distributions (column/locator-to-variable linkage). This now applies to BOTH sibling DataDownload distributions AND to `schema:potentialAction.schema:result` inside a WebAPI distribution.
- `cdi:TabularTextDataSet`, `cdi:LongStructureDataSet`, or `cdi:StructuredDataSet` typing on a distribution OR on a WebAPI's Action.result
- `csvw:` properties (delimiter, header, headerRowCount)

Records that only have `cdi:InstanceVariable` in `@type` without any of the above should go to:
- **Discovery** repo — if they have spatialCoverage/temporalCoverage/variableMeasured
- **packaging** repo — if they are archive bundles (hasPart with multiple files)

Records that additionally use `cdi:isStructuredBy` with a `cdi:DataStructure` graph, `cdi:RepresentedVariable` nodes, `cdi:has_DataStructureComponent`, `cdi:IdentifierComponent` / `MeasureComponent` / `AttributeComponent` / `DimensionComponent` / `VariableValueComponent` / `VariableDescriptorComponent`, or DataStructure subtypes (`cdi:DimensionalDataStructure`, `cdi:LongDataStructure`, `cdi:WideDataStructure`) belong in the **data-structure-description** profile, not here. The Data-Structure profile composes the Data-Description profile and adds the full DDI-CDI structural complexity.

## Example sources

### DDI Codebook conversion (primary source)

Harvard Dataverse provides DDI Codebook 2.5 with variable-level descriptions for ingested .tab files:

```bash
# Download DDI
curl -o dataset.xml "https://dataverse.harvard.edu/api/datasets/export?exporter=ddi&persistentId=doi:10.7910/DVN/XXX"

# Convert to CDIF DataDescription with physical mappings, file metadata
python validation/DDI/ddi_to_cdif.py dataset.xml \
  --doi https://doi.org/10.7910/DVN/XXX \
  --fetch-headers --fetch-file-meta \
  -o examples/ddi-my-dataset-datadesc.json
```

The DDI-to-CDIF mapping:
- DDI `<var name>` → `schema:name`
- DDI `<labl>` → `schema:description`
- DDI `<varFormat type>` → `cdi:intendedDataType` (numeric→xsd:decimal, character→xsd:string)
- DDI `<var intrvl>` → `cdi:role` (contin→MeasureComponent, discrete→AttributeComponent)
- DDI `<sumStat min/max>` → `schema:minValue`/`schema:maxValue`
- DDI `<caseQnty>` → `cdifq:nRows`
- DDI `<varQnty>` → `cdifq:nColumns`
- Tab file headers → `cdi:hasPhysicalMapping` with column index and `cdi:formats_InstanceVariable`
- Dataverse file API → `schema:contentSize`, `spdx:checksum`

## Synced files

These are copied / generated from metadataBuildingBlocks and should be updated when the source changes:

- `CDIFDataDescriptionProfileStructuredSchema.json` ← byte-synced from `metadataBuildingBlocks/_sources/profiles/cdifProfiles/CDIFDataDescriptionProfile/resolvedSchema.json` (generated by `tools/resolve_schema.py CDIFDataDescriptionProfile`).
- `CDIFDataDescription-frame.jsonld` — hand-maintained JSON-LD frame used by `FrameAndValidate.py`. Mirrors the resolved schema's property tree; keep the `cdif`/`cdi` namespace prefixes correct (in particular: `cdif:hasPhysicalMapping` / `cdif:fileSize` / `cdif:fileSizeUofM` / `cdif:formats_InstanceVariable` are `cdif:` not `cdi:`).
- `dataDescriptionRules.shacl` — **merged** from 15 source SHACL files (see README's [SHACL Validation](README.md#shacl-validation) section for the source list). The merge uses shape-name precedence dedup: when the same named SHACL shape appears in multiple sources, the later (higher-precedence) source wins. Regenerate via the in-repo merge script after source SHACL changes.
- `CDIFDataDescriptionImplementationGuide.docx` ← regenerated from `.md` via `pandoc --reference-doc=<prior committed .docx>` to preserve Word styles.

## Validation

```bash
python -c "
import json
from jsonschema import Draft202012Validator
schema = json.load(open('CDIFDataDescriptionProfileStructuredSchema.json'))
doc = json.load(open('examples/ddi-cost-of-cash-datadesc.json'))
errors = list(Draft202012Validator(schema).iter_errors(doc))
print(f'{len(errors)} errors')
"
```

## Key namespaces

| Prefix | IRI | Usage |
|--------|-----|-------|
| `cdi` | `http://ddialliance.org/Specification/DDI-CDI/1.0/RDF/` | `InstanceVariable`, `RepresentedVariable`, `TabularTextDataSet` / `StructuredDataSet` / `PhysicalDataSet`, `DataStructure*`, `characterSet`, `function`/`platformType`/`source`/`hasIntendedDataType`/`describedUnitOfMeasure`/`qualifies`, `takesSentinelValuesFrom`/`takesSubstantiveValuesFrom`, `Statistics`, `CategoryStatistics`, `ComponentPosition`, `indexes`/`value`, `for`, etc. |
| `cdif` | `https://cdif.org/0.1/` | `physicalDataType`, `role`, `simpleUnitOfMeasure`, `uses`, `hasPhysicalMapping`, `formats_InstanceVariable`, `index`, `locator`, `format`, `fileSize`, `fileSizeUofM`, `hasPrimaryKey`, `Key`, `isComposedOf`, `statistics`, `isDescribedBy_StatisticsCollection`, `StatisticsCollection`, `has_Statistics`, `appliesTo`, `indexedBy`, `has_CategoryStatistics`, `SubstantiveValueDomain`, `SentinelValueDomain`, `EnumerationDomain`, `takesValuesFrom`, `recommendedDataType`, `references`, `displayLabel`. **Note**: declare the `cdif` prefix in `@context` — it is easy to miss because schema.org examples often only declare `cdi`; without it `cdif:*` keys are treated as relative IRIs and silently fail SHACL/frame matching. |
| `csvw` | `http://www.w3.org/ns/csvw#` | `delimiter`, `header`, `headerRowCount`, `quoteChar`, `skipRows`, etc. |
| `spdx` | `http://spdx.org/rdf/terms#` | `checksum`, `algorithm`, `checksumValue` |
| `skos` | `http://www.w3.org/2004/02/skos/core#` | `Concept`, `ConceptScheme`, `prefLabel`, etc. — used inline inside `cdif:EnumerationDomain.cdif:references` |
| `dqv` | `http://www.w3.org/ns/dqv#` | `QualityMeasurement`, `isMeasurementOf`, `value` |
